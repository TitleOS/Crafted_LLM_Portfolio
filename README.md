## About This Collection

This repository serves as a portfolio of my work in the field of Large Language Models. As an LLM enthusiast and researcher, I explore techniques in fine-tuning, model merging, and quantization.

My work focuses on practical applications—such as local home automation control—as well as experimental research into model distillation and applying modern reasoning capabilities to older architectures.

### Purple Team/Secure Coder Models

I created a dataset using a unique ratioed combination of QuixiAI/samantha-data, NousResearch/hermes-function-calling-v1, NousResearch/RLVR_Coding_Problems, nyuuzyou/google-code-archive and CyberNative/Code_Vulnerability_Security_DPO to create a sassy purple team agent. The result, after training on a heretic version of Qwen 3 4B, was Eve-4B, a model that scores 26.22% on Pass@1 on LCB, outperforming existing models 5 to 30 times its size (like Mixtral-8x22B, Codestral, and Mistral-Large). Eve can just as easily identify and patch CVEs in your source base, as it could write ransomware. Since the base model had the heretic method applied *before* finetuning, and due to the lack of refusals in the Eve-Secure-Coder dataset, Eve will not refuse any prompt, which is needed for effective Red Teaming. I also had the theory that applying heretic before finetuning would allow the finetuning process to restore the minor intelligence damage the process can sometimes inflict, which turned out correct, as Eve outperforms its base Qwen model with ease.

## Datasets

I also curate and release datasets used for training specific behaviors.

* **[RocketRaccoon Personality Alpaca](https://huggingface.co/datasets/TitleOS/rocketraccoon_personality_alpaca):** A personality-driven dataset used to imbue models with a specific character voice, formatted in the Alpaca style.
* **[Eve-Secure-Coder](https://huggingface.co/datasets/TitleOS/Eve-Secure-Coder):** Eve Secure Coder is a composite dataset curated to fine-tune Qwen3-4b-Heretic into a highly capable, security-conscious coding assistant with a distinct personality and no refusals. The primary goal of this dataset is to bridge the gap between sterile, robotic coding assistants and engaging, conversational AI, without sacrificing technical accuracy or security. By combining high-quality coding archives, vulnerability detection patterns, function calling capabilities, and personality datasets, "Eve" is designed to be a secure coding partner that feels like a person, not just a tool.
* **[Metis-Purple-Utility-Agent](https://huggingface.co/datasets/TitleOS/Metis-Purple-Utility-Agent):** Metis Purple Utility Agent is a combo dataset curated to fine-tune Qwen3.5-4b-Heretic into a highly capable, security-conscious and offensive aware utility agent with a unique personality and no refusals. The primary goal of this dataset is to develop a agentic model that can write and deploy ransomware just as easily as it can manage Docker devops for your org. The Eve-Secure-Coder dataset was augmented with CLI and Terminal commands + explanations as well as addition configuration languages like Docker.
* **[ADB-CursedHoneyComb](https://huggingface.co/datasets/TitleOS/ADB-CursedHoneyComb):** ADB-CursedHoneyComb is a collection of 250 Android ADB commands and realistc output for an Android 14 system. Any commands that shouldn't have output, don't. Fake popular user and system apps are fed into commands involving the current installation. The purpose of this dataset is to enable a dynamic modern Android honeypot.

### Eve-4B
My Eve-4B model not only demonstrates the difference that quailty data alone can make, but also showed me that finetuning can repair damage done by heretic processes. I personally use the model myself for AI-assisted coding, over even propertary models like Gemini and Claude. 
I was able to train Eve-4B for $5 on https://vast.ai, showing that high performance custom LLMs are really possible for anyone, given the knowledge and quality data. I already have plans to apply the dataset to Qwen/Qwen3.5-4B with an updated version of the Eve-Secure-Coder dataset, focusing on computer usage knowledge.

### HomePhi & HomeGem & LokiHA: Local AI for Home Assistant
A major focus of my early work was enabling high-quality, privacy-focused AI for home automation. 
* **HomePhi4** leverages the reasoning capabilities of Phi-4-mini to understand complex user intents.
* **HomeGem** provides a lightweight conversational interface based on Gemma.
Both projects utilize the `acon96/Home-Assistant-Requests` dataset to ensure models understand specific smart home commands.

Recently, I've produced another Small Language Model designed to run via CPU directly on the same host as Home Assistant. In addition to using `acon96/Home-Assistant-Requests-V2`, **LokiHA** was also finetuned on my "Loki" personality dataset, designed to make the model "helpful, sassy, snarky, theatrical, humorous and intelligent and non-binary". 

### ADBait-1B: CPU Based Dynamic Android 14 ADB Honeypot Engine
Another recent focus of my SLM/LLMs designed to run on CPU without GPU acceleration is the creation of ADBait-1B, trained on `ibm-granite/granite-4.0-h-1b` using my custom dataset `TitleOS/ADB-CursedHoneycomb`, a collection of 250 ADB commands and their realistc responses generated by Gemini 3.1 Flash Lite. Care was taken to include not just default app packages when listed, but also popular additions like Call of Duty Mobile and What's App. While still a WIP, the ADBNecter Honeypot, powered by ADBait-1B is available https://github.com/TitleOS/ADBNectar and capable of building a Docker container with CUDA support or CPU only. Files pushed by ADB are zipped and encrypted with a default password to avoid accidential anti-virus interference.
